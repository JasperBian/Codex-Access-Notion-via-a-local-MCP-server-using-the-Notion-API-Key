# Notion Research Documentation

Use this skill when documenting research, technical exploration, or reference material into a Notion workspace.

## Workflow

### 1. Structure research documents
Organize research notes with these sections:

- **Executive Summary**: 2-3 sentence overview of findings
- **Background**: Context and motivation for the research
- **Methodology**: How the research was conducted
- **Findings**: Detailed findings with evidence
- **Conclusions**: Key takeaways and recommended next steps
- **References**: Links to sources, related pages, and code

### 2. Format for readability
- Use **toggle headings** for detailed sections that can be collapsed
- Use **callout blocks** for key insights or warnings
- Use **code blocks** with language annotations for technical snippets
- Use **numbered lists** for sequential processes
- Use **bulleted lists** for non-ordered items
- Use **database views** for comparing multiple options

### 3. Cross-reference related content
- Link to related Notion pages and databases
- Mention linked issues, PRs, or tickets
- Tag pages with relevant keywords for searchability
- Reference source code by file path and line numbers

## Guidelines

- Be thorough but organized — use progressive disclosure (toggles for detail)
- Use Notion's native tables for structured comparison data
- Include timestamps for time-sensitive findings
- When comparing options, use a database with properties for side-by-side analysis
- Always link back to source material
