# Notion Knowledge Capture

Use this skill when you need to save important information, insights, or structured knowledge into a Notion workspace.

## Workflow

### 1. Identify what to capture
- Key decisions made during a conversation
- Research findings or code exploration results
- Architecture decisions and trade-off analysis
- Important links, references, or resources

### 2. Choose the right Notion container
- **Page**: For standalone notes, decisions, or reference docs
- **Database row**: For structured data that belongs in an existing database (tasks, projects, issues)
- **Existing page append**: For adding notes to an existing reference page

### 3. Use MCP tools to create content

**Create a new page:**
The MCP server provides tools to create pages under a parent. When asked, use the appropriate tool to create well-structured Notion pages with headings, bullet lists, callouts, and code blocks.

**Append to existing page:**
When adding to an existing page, use the append blocks tool. Structure additions clearly with headings and organized sections.

**Insert database rows:**
When working with structured data, identify the target database and insert properly formatted rows.

## Guidelines

- Use rich formatting: headings, bullet lists, numbered lists, callouts, code blocks, and toggles
- Add meaningful titles that make pages searchable
- Link related pages together when appropriate
- For technical content, use code blocks with language annotations
- Keep pages focused — one topic per page
- Use Notion databases for data that needs filtering, sorting, or grouping
