# Notion Meeting Intelligence

Use this skill when capturing meeting notes, summarizing discussions, tracking action items, or organizing meeting-related content in Notion.

## Workflow

### 1. Create meeting notes page
Structure with:
- **Meeting title**: `[Type] Topic - Date`
- **Date, attendees, tags** as database properties
- **Agenda**: planned discussion points
- **Notes**: real-time or summarized notes organized by topic
- **Decisions**: clear statements of what was decided
- **Action items**: owner + deadline for each item

### 2. Capture action items
For each action item include:
- Clear description of the task
- Assigned owner
- Due date or timeframe
- Priority level
- Link back to the meeting notes

### 3. Link to existing databases
- Connect meeting notes to a Meetings database if one exists
- Create or update task entries in a Tasks/Projects database
- Link decisions to relevant project pages

## Guidelines

- Create meeting notes in a dedicated Meetings database when available
- Use the same template consistently across meetings
- Tag meetings by project, team, or topic for filtering
- Ensure every decision has a clear owner
- Follow up on action items in subsequent meetings
